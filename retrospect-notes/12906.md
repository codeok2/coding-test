# 📝 [Java] List & 배열 타입 변환 핵심 요약 노트

## 1. List 선언과 초기화 (`new List<>()`가 안 되는 이유)

- **결론**: `List`는 인터페이스(설계도)이므로 `new List<>()`로 객체를 직접 생성할 수 없음.
- **올바른 사용**: 실제 구현체인 클래스를 지정해야 함.

```java
List<Integer> list = new ArrayList<>();  // 가장 일반적
List<Integer> list = new LinkedList<>(); // 삽입/삭제 빈번할 때
```

---

## 2. 변수 선언 후 나중 초기화 가능 여부

- **지역 변수 (메서드 안)**: 선언만 하는 건 가능하나, 초기화 없이 사용(읽기)하면 컴파일 에러 발생. (자바가 기본값을 안 채워줌)
- **멤버 변수 (클래스 직하)**: 선언만 해도 초기화 없이 사용 가능. 자바가 객체 생성 시 타입별 기본값(`0`, `false`, `null`)으로 자동 초기화해 줌.

---

## 3. List ➔ 배열 변환 3가지 방법

```java
List<Integer> list = new ArrayList<>(List.of(10, 20, 30));

// ① Integer[] 로 변환 시 (가장 추천)
Integer[] arr1 = list.toArray(new Integer[0]);

// ② int[] 로 변환 시 (전통적 / 성능 최고)
int[] arr2 = new int[list.size()];
int index = 0;
for (int num : list) arr2[index++] = num;

// ③ int[] 로 변환 시 (코테 필수 / 간결함)
int[] arr3 = list.stream().mapToInt(Integer::intValue).toArray();
```

---

## 4. `list.toArray(new Integer[0])`의 비밀

- 인자로 크기가 0인 빈 배열(`new Integer[0]`)을 넘겨주면, 자바 내부에서 리스트 개수(`list.size()`)를 확인하고 알아서 딱 맞는 크기의 배열을 새로 생성해서 값을 채워 반환함.
- 최신 자바 버전에서는 크기를 미리 지정하는 것보다 `new Integer[0]`을 쓰는 것이 코드도 깔끔하고 성능도 동등하거나 더 우수함.

---

## 5. ★★★ 가장 중요한 핵심: `int[]`와 `Integer[]` 변환의 한계

- **단일 변수** (`int` ↔ `Integer`): 자바가 오토박싱/언박싱을 해주므로 자동 변환 가능.
- **배열 변수** (`int[]` ↔ `Integer[]`): 배열은 독립된 객체이므로, 두 배열은 뿌리(족보)가 아예 다른 남남임. 상속 관계도 아님.

**메모리 구조 차이**

| 타입 | 메모리 구조 |
|---|---|
| `int[]` | 실제 숫자 데이터가 다이렉트로 들어있음 |
| `Integer[]` | 실제 숫자가 아닌, 숫자가 저장된 객체의 주소값들이 들어있음 |

**결론**: 구조가 완전히 다르기 때문에 `list.toArray(new int[0])` 같은 문법은 존재할 수 없음 (제네릭 제한 + 타입 불일치). `int[]`로 바꾸려면 무조건 스트림이나 `for`문으로 알맹이를 하나씩 꺼내서(언박싱) 옮겨 담아야 함.

---

## 6. 코딩 테스트 스트림(Stream) 사용 가이드

- **스트림을 쓰는 이유**: 성능 때문이 아니라 오직 코드의 간결함과 가독성(생산성) 때문임. (한 줄 컷 및 체이닝 연산이 매우 강력함)
- **성능 주의**: 스트림은 내부 오버헤드가 있어서 `for` 루프나 `toArray()`보다 느림.
- **코테 꿀팁**: 대부분의 문제는 스트림을 써도 통과하지만, 만약 효율성 테스트에서 아슬아슬하게 시간 초과(TLE)가 난다면 가장 먼저 스트림 코드를 일반 `for`문으로 바꿀 것!
