## ✅ 스스로 잡았던 것

- **시간복잡도 감각**: `phone_book` 최대 길이 100만 → `O(n²)`(이중 for문)이면 시간 초과 위험. `O(n log n)`이면 약 17 × 100만 ≈ 1초 이내라 안전하다고 판단.
- 이 판단이 **정렬 접근으로 가는 출발점**이 됐다. 복잡도부터 따지고 알고리즘을 고르는 흐름은 유지하자.

## 💡 새로 알게 된 것 (질문별)

### 1. "정렬하면 접두어 관계인 번호끼리 인접한다"

- 처음엔 뒤죽박죽 배열에서 모든 쌍을 비교해야 한다고 생각 → `O(n²)`.
- 사전순 정렬하면 `"12"`, `"123"`, `"1235"`처럼 **접두어 관계인 번호들이 서로 붙는다.**
- 그래서 전체 비교가 아니라 **인접한 원소끼리만** 확인해도 충분 → 비교가 `O(n)`으로 줄어듦.
- 👉 교훈: **정렬은 "비교해야 할 범위를 좁혀주는" 도구다.**

### 2. 이건 "슬라이딩 윈도우"가 아니다 (용어 정정)

- 내가 처음에 "윈도우 사이즈 2짜리 슬라이딩 윈도우"라고 불렀는데 **틀린 용어였다.**
- **슬라이딩 윈도우** = 연속 구간의 합·개수·최댓값 같은 **"유지되는 누적 상태"**를 들고, 윈도우를 밀 때 빠지는 값/들어오는 값만 갱신해서 재계산을 아끼는 기법.
- 이 문제는 그런 누적 상태 없이 **매번 독립적으로 두 개만 비교** → 정확한 이름은 **"정렬 후 인접 원소 비교"**.
- 👉 교훈: 면접에서 용어 잘못 쓰면 꼬리질문에서 어긋난다. 패턴 이름을 정확히 부르자.

### 3. 해시 풀이 — "관점을 뒤집기"

- 내 첫 발상: **"이 원소가 남의 접두어인가?"** → 비교할 더 긴 원소를 찾아야 해서 막혔다. (가장 긴 원소를 찾아야 하나? 고민)
- 뒤집은 발상: **"내 접두어들이 전화번호부에 존재하나?"**
- 번호 하나(`"1195524421"`)의 접두어 조각(`"1"`, `"11"`, `"119"`, ...)을 직접 만들어서, 그게 `HashSet`에 있는지 `contains()`로 확인.
- `Set`은 순서가 없어 정렬·인접 비교는 안 되지만, **"이거 있어?"라는 질문에 `O(1)`로 답하는 게 진짜 강점.**
- 👉 교훈: 막히면 **"찾는 방향을 반대로 뒤집을 수 있나?"**를 떠올리자.

### 4. 자기 자신은 접두어에서 빼야 한다

- 접두어를 만들 때 전체 문자열(자기 자신)까지 포함하면, 자기가 전화번호부에 있으니 **무조건 `false`가 되는 함정.**
- `i`를 `1`부터 `length - 1`까지만 잘라서 (`substring(0, i)`, 조건 `i < number.length()`) 자기 자신은 제외.

### 5. 두 반복문을 왜 분리해야 하는가

- `add`와 `contains`를 **한 루프에서 동시에** 하면 순서에 따라 놓칠 수 있다.
- 예: `["119", "11"]`에서 `"11"`을 아직 안 넣은 상태로 `"119"`의 접두어를 검사하면 → `"11"`을 못 찾고 지나침.
- **모든 번호를 Set에 다 넣은 뒤 검사**해야 입력 순서와 무관하게 정확하다.
- 👉 교훈: "채우기 → 검사하기" 단계를 나눈 데는 이유가 있다. 무의식적으로 나누지 말고 이유를 알고 나누자.

### 6. 제한사항이 곧 알고리즘 힌트였다

- 전화번호 길이 ≤ 20 → 접두어는 번호당 최대 19개 → 해시 풀이 복잡도는 `O(n × 20) = O(n)`.
- 이 "길이 20 제한"이 해시 풀이를 가능하게 한 **숨은 조건**이었다.
- 👉 교훈: 제한사항을 그냥 읽지 말고 **알고리즘 선택의 근거로** 읽자.

## ⚖️ 두 풀이 트레이드오프 (면접 대비)

| 접근 | 시간복잡도 | 공간 | 특징 |
|---|---|---|---|
| 정렬 후 인접 비교 | `O(n log n)` | 추가 자료구조 없음 | 코드가 짧고 직관적 |
| 해시(Set) | `O(n)` | Set 메모리 사용 | 더 빠르지만 메모리 트레이드오프 |

> "다른 방법으로도 풀 수 있나요?" 꼬리질문 대비 한 문장:
> **정렬은 추가 자료구조 없이 간단하지만 `O(n log n)`, 해시는 Set 메모리를 쓰는 대신 `O(n)`.**

## 🧑‍💻 내 풀이

### 정렬 풀이

```java
import java.util.*;

class Solution {
    public boolean solution(String[] phone_book) {
        Arrays.sort(phone_book);

        for (int i = 0; i < phone_book.length - 1; i++) {
            if (phone_book[i + 1].startsWith(phone_book[i])) {
                return false;
            }
        }
        return true;
    }
}
```

### 해시 풀이

```java
import java.util.Set;
import java.util.HashSet;

class Solution {
    public boolean solution(String[] phone_book) {
        Set<String> set = new HashSet<>();
        for (String number : phone_book) {
            set.add(number);
        }

        // 본인 접두어가 집합에 있는지 (해시 활용)
        for (String number : phone_book) {
            for (int i = 1; i < number.length(); i++) {
                String part = number.substring(0, i);
                if (set.contains(part)) {
                    return false;
                }
            }
        }
        return true;
    }
}
```
